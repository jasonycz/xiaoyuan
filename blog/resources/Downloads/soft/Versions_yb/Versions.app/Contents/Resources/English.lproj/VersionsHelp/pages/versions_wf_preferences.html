Preferences

Timeline preferences

At the top of the preferences window, you can set how many revisions are loaded into the Timeline when you open or refresh it. It's always possible to load more revisions while you are viewing the Timeline, so you can keep this number low to make sure the Timeline loads as quickly as possible.

File Browser preferences

With this feature switched on, Versions will listen for file system notifications, automatically refreshing the display of the browser view when something changes inside your working copy. Basically, the status you see in Versions will then always be up to date.

File Comparison preferences

Use the drop-down menu next to “File Comparison:” to select which application or script Versions should use to compare changes between different versions of files. See the section: Comparing differences for more information about file comparisons, which external applications are supported and how to get them, and how to create a custom comparison script.

Bookmarks List preferences

Versions shows a badge next to each working copy bookmark to indicate the number of local and remote changes to that working copy. The data to display these badges is fetched every once in a while in the background. If you have a large number of working copies visible in your bookmarks list, switching off “Show badges with working copy changes” may improve performance and reduce network traffic.

Notifications preferences

A notification will be displayed on screen once an action is completed, such as updates or commits.

Software Updates preferences

Versions can automatically check for updates and notify you when they are available. Including your anonymous system profile gives us a better understanding of our user base.

Proxy preferences

Versions allows you to specify a custom proxy server to use for http(s) connections. To use a custom proxy server, enable the "Use proxy for HTTP(S) connections" option and fill in the details of your server in the proxy configuration sheet.

Subversion Library preferences

Versions supports Subversion 1.4 through 1.8. Subversion 1.7 is selected by default. If you would like to be able to use Versions alongside Xcode SCM integration or other Subversion clients, this default setting is probably the best option.
