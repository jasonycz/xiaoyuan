Comprendre l'affichage de Comparaison de Dossiers

L'affichage de Comparaison de Dossiers utilise des signaux de couleur pour mettre en évidence les différences entre les deux dossiers de base.  Par défaut, les significations de couleur sont :

 

Gris

Inconnu ou plus ancien

 

Noir

Identique

 

Pourpre

Orphelin (n'existe pas de l'autre côté)

 

Rouge

Plus récent ou différent

Sélectionner Affichage > Légende pour afficher les diverses combinaisons de couleur de dossier.

Pour changer les couleurs par défaut, sélectionnez Outils > Options , allez à la page Couleurs, Fontes > Vues Dossier, et trouvez les paramètres de Couleurs de comparaison.

Vous pouvez cliquer sur une entête de colonne pour trier sur cette colonne, ou cliquer sur l'entête avec le bouton droit pour sélectionner d'autres colonnes à afficher.

Dans l'exemple suivant, le fichier dans le côté droit est coloré en rouge pour montrer qu'il est plus récent.  L'icône dans la colonne centrale indique qu'une comparaison de contenu a trouvé des différences.

La marque de couleur à gauche d'un nom de fichier indique un signal de couleur pour une différence, même si la sélection obscurcit la coloration des autres éléments de la liste.

La colonne centrale affiche les résultats d'une comparaison de contenu, si une a été exécutée.

Les résultats d'une comparaison de contenu peuvent passer outre des couleurs de fichier.  Par exemple, la ligne ci-dessus serait changée en noire si la comparaison du contenu avait déterminé une correspondance de fichier.  Pour désactiver ce comportement, sélectionnez Session > Paramètres de session , allez à l'onglet Comparaison, et décochez la case Ne pas tenir compte des résultats des tests rapides.

Sur les systèmes de fichier Windows, le nom de fichier peut être suivi par des lettres entre parenthèses qui représentent les attributs DOS de lecture seule (r), caché (h), archive (a) et système (s).

Quelques types de fichier, tels que les fichiers archive zip, peuvent contenir plusieurs fichiers.  Beyond Compare peut les traiter comme des dossiers normaux.  L'exemple suivant affiche notre fichier d'aide compilé avec une version plus récente du côté droit.  Notez l'icône spéciale "fermeture éclair" ("zipper" en anglais).

Dossiers dans l'affichage

Les dossiers sont colorés pour vous donner une vue sur leur contenu.  Par exemple, la comparaison ci-dessous vous indique que les dossiers contiennent au moins un fichier qui est plus récent à droite (rouge) et que les deux côtés ont au moins un fichier orphelin (pourpre).

Les dossiers peuvent apparaître évidés lorsque vous commencez la comparaison.  Ceci arrive lorsque le traitement de balayage en tâche de fond n'a pas encore complété ces dossiers.  Ils changeront de couleur lorsque le traitement sera fini.

Un dossier sera jaune si le balayage automatique n'est pas enclenché et que vous n'avez pas encore ouvert le dossier.  Une fois que vous avez manuellement ouvert le dossier (et éventuellement ses sous-dossiers) il sera coloré de manière adéquate.

Les icônes de dossier incluront une petite flèche incrustée si le dossier est un point de jonction ou un lien symbolique.  Les icônes incluront un petit "X" rouge incrusté s'il y a une erreur en essayant d'ouvrir le dossier.  L'exemple suivant affiche un dossier avec les deux conditions.
